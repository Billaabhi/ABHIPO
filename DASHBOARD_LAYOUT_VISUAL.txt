KEY INDICATORS:

Status Icons:
  ✓  = Done
  ⟳  = In Progress
  ⚪  = To Do
  ✓✓ = Completed

Color Codes:
  🟢 Green (70%+)      = On track
  🟡 Yellow (40-70%)   = At risk
  🔴 Red (<40%)        = Needs help
  🟣 Purple            = Data field

Workload Indicators:
  → Busy!              = High workload
  → Very busy!         = Critical overload
  → Has capacity       = Available

Auto-Updates:
  🔄 Every 60 seconds from Jira
  No manual refresh needed


WHAT EACH SECTION TELLS YOU:

1. SPRINT SUMMARY
   ├─ Are we on track? (Completion % should be 40-60% mid-sprint)
   ├─ What's the WIP? (In Progress should be 5-8 ideally)
   └─ How much left? (To Do items decreasing over sprint)

2. EPICS IN PROGRESS
   ├─ Which features are on track? (Green bars)
   ├─ Which are at risk? (Yellow/red bars)
   └─ How's the breakdown? (Done vs pending issues)

3. INITIATIVES & ROADMAP
   ├─ Strategic progress (labeled items by initiative)
   ├─ What's our priority? (items in current sprint)
   └─ When will it ship? (completion % → days left)

4. TEAM ASSIGNMENTS & WORKLOAD
   ├─ Who's busiest? (highest story points)
   ├─ Who has capacity? (lowest story points)
   └─ Are we balanced? (compare numbers across team)

5. CRITICAL ISSUES & BLOCKERS
   ├─ What's blocking us? (red blockers)
   ├─ Who can fix it? (assigned engineer)
   └─ How long? (estimate from status)


DAILY USAGE PATTERN:

Morning Standup (10:00 AM):
  1. Open POD Dashboard
  2. Glance at completion % → "Are we on track?"
  3. Scan epics → "What's at risk?"
  4. Check blockers → "What needs help?"
  5. Review assignments → "Is anyone overloaded?"

Decision Making:
  → Red blocker? → Escalate to PO
  → Engineer overloaded? → Reassign work
  → Epic at risk? → Add resources
  → All green? → Keep momentum

Weekly Planning:
  → Compare completion % to previous week
  → Check epic trend (getting better?)
  → Review assignment distribution
  → Plan next sprint with this data
